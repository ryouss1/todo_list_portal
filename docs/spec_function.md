## 1. 機能一覧

| 機能カテゴリ | 機能 | 説明 |
|-------------|------|------|
| **認証** | ログイン | メールアドレスとパスワードでセッション認証 |
| | ログアウト | セッション破棄、ログイン画面へリダイレクト |
| | アクセス制御 | 未認証ユーザーはログイン画面へリダイレクト、API は 401 |
| | RBAC | admin/user ロールによる権限制御 |
| | パスワードポリシー | パスワード作成・変更時に強度要件を検証（最小8文字、大文字/小文字/数字必須） |
| | レート制限 | ログイン失敗を15分間に5回までに制限（設定可能） |
| | アカウントロックアウト | 連続ログイン失敗でアカウントを30分間ロック、管理者がアンロック可能 |
| | セッション無効化 | パスワード変更・ロール変更時に既存セッションを強制失効 |
| | 監査ログ | ログイン、パスワード変更、ロール変更等の認証イベントを記録・閲覧 |
| | OAuth2/SSO連携 | Google/GitHubアカウントでのログイン、既存アカウントとの自動リンク |
| | OAuthプロバイダ管理 | 管理者によるOAuthプロバイダの登録・更新・削除・有効/無効切替 |
| | OAuthアカウントリンク | ユーザーが自分のアカウントにOAuthアカウントをリンク・アンリンク |
| | パスワードリセット | メールアドレスによるトークンベースのパスワードリセット（SMTP送信） |
| **Todo管理** | Todo作成 | タイトル、説明、優先度（Normal/High/Urgent）、期日を指定して作成 |
| | Todo更新 | 各フィールドの部分更新に対応 |
| | Todo削除 | 確認ダイアログ後に削除 |
| | 完了トグル | チェックボックスで完了/未完了を切り替え |
| | フィルタリング | All / Active / Completed でTodo一覧を絞り込み |
| | 公開範囲 | private/public の設定、公開Todo一覧画面 |
| **出勤管理** | 出勤記録 | メモ付きで出勤を記録（二重出勤不可） |
| | 退勤記録 | メモ付きで退勤を記録（未出勤時は不可） |
| | 出勤状態確認 | 現在出勤中かどうかをリアルタイム表示 |
| | 経過時間表示 | 出勤中の経過時間をリアルタイムカウントアップ |
| | 出勤履歴 | 過去の出勤・退勤記録を一覧表示（月別フィルタ対応） |
| | 休憩管理 | 1回の出勤あたり最大3回まで休憩の開始・終了を記録 |
| | 手動入力 | 日付・出勤退勤時刻を指定して勤怠記録を手動作成 |
| | プリセット | 定時パターン（出勤・退勤・休憩時刻）の設定・適用（手動作成・編集モーダルの両方で利用可能） |
| | Excelエクスポート | 月別の勤怠データをExcelファイルでエクスポート |
| | 入力種別ロック | input_type=admin の記録は編集・削除不可 |
| **タスク管理** | タスク作成 | タイトル、説明、タスク分類、Backlogチケット番号を指定して作成 |
| | タスク更新 | タイトル、説明、ステータス、タスク分類、Backlogチケット番号の更新 |
| | Backlogチケット連携 | タスクにBacklogチケット番号（例: WHT-488）を登録し、URLリンクを生成 |
| | タスク削除 | 確認ダイアログ後に削除（関連タイムエントリもCASCADE削除） |
| | タイマー開始 | タスクの作業時間計測を開始（ステータスが自動でin_progressに変更） |
| | タイマー停止 | タイマーを停止し経過秒数を累計に加算 |
| | 時間表示 | 累計作業時間をHH:MM:SS形式でリアルタイム表示 |
| | タスク完了 | Done操作でタスクを削除（report=true時は日報自動作成） |
| | 一括完了 | 複数タスクを一括完了（Batch-Done） |
| **タスクリスト管理** | アイテム作成 | タイトル、説明、予定日、カテゴリ、Backlogチケット番号を指定して作成 |
| | アイテム更新 | 各フィールドの部分更新、ステータス変更に対応 |
| | アイテム削除 | 確認ダイアログ後に削除 |
| | 担当割り当て | 未割当アイテムを自分に割り当て |
| | 担当解除 | 割り当て済みアイテムを未割当に戻す（公開プールに返却） |
| | タスク開始 | アイテムをTasksにコピーしてタイマー管理を開始 |
| | 時間蓄積 | Task完了時に作業時間をソースアイテムに蓄積 |
| | フィルタ | ステータス・カテゴリ・キーワードでフィルタリング（done はデフォルト非表示） |
| **在籍管理** | ステータス更新 | 自分の在籍状態を更新（available/away/out/break/offline/meeting/remote） |
| | 全員の状態表示 | 全ユーザーの在籍状態を一覧表示（display_name付き） |
| | 作業中チケット表示 | in_progressタスクのBacklogチケットを在籍一覧に表示 |
| | リアルタイム配信 | WebSocketでステータス変更をリアルタイムにブラウザへ配信 |
| | 変更履歴 | 自分の在籍状態変更履歴を取得 |
| **日報管理** | 日報作成 | 対象日、タスク分類、タスク名、作業時間、業務内容等を指定して作成 |
| | 日報更新 | 所有者のみ各フィールドの部分更新に対応 |
| | 日報削除 | 所有者のみ削除可能 |
| | 全員の日報閲覧 | 全認証ユーザーが全ユーザーの日報を閲覧可能 |
| | タスク完了連携 | Task Done時にreport=trueのタスクから日報を自動作成 |
| **業務サマリー** | 期間別集計 | 日次/週次/月次で業務サマリーを集計・表示 |
| | 日報提出状況 | ユーザーごとの日報提出状況を表示 |
| | カテゴリ別集計 | タスク分類別の作業時間・件数を集計 |
| | 課題集約 | 期間内の課題・問題を集約表示 |
| **ログ管理** | ログ登録 | 外部システムからAPI経由でログを登録 |
| | ログ一覧 | 受信日時降順でログを一覧表示（件数上限指定可） |
| | 重要ログフィルタ | WARNING/ERROR/CRITICALのログのみ表示 |
| | リアルタイム配信 | WebSocketでログをリアルタイムにブラウザへ配信 |
| **ログソース管理** | ログソース作成 | リモートサーバー接続情報（FTP/SMB）、パーサーパターン等を指定して作成（管理者操作） |
| | ログソース更新 | 各フィールドの部分更新に対応（管理者操作） |
| | ログソース削除 | 管理者のみ削除可能 |
| | 接続テスト | リモートサーバーへの接続テストを実行し、ファイル一覧取得を検証（管理者操作） |
| | ファイル一覧 | ログソースに紐づく収集対象ファイルの一覧表示 |
| | バックグラウンド収集 | リモートサーバーへのポーリングによる自動ログ収集（`LOG_SCANNER_ENABLED=true` で有効化、各ソースの `polling_interval_sec` に基づく自動スキャン） |
| | スキャン実行 | リモートサーバーに接続してファイル一覧を取得し、当日のファイルのみDB更新（管理者操作） |
| | コンテンツ再読込 | 既存のlog_entriesを削除しlast_read_lineをリセットした後、再スキャンを実行（エンコーディング変更後の再取得用、管理者操作） |
| | ファイル変更通知 | alert_on_change有効時、スキャンでファイルのタイムスタンプ更新を検出し、画面に「エラーあり」表示 + 変更ファイル名 + フォルダリンクを表示。アラートも自動生成（変更ファイル名・フォルダパス含む） |
| | フォルダリンク | 変更検出時にフォルダパスを表示（SMB: `\\host\path`、FTP: `ftp://host/path`）。クリックでパスをクリップボードにコピー（ブラウザのセキュリティ制約により `file://` リンクは動作しないため） |
| **アラート管理** | アラート一覧 | アラートの一覧表示（アクティブ/重要度/ソース/キーワードフィルタ対応） |
| | 手動アラート作成 | タイトル、メッセージ、重要度を指定して手動作成 |
| | アラート確認 | 確認済みにマーク（確認者・日時を記録） |
| | アラート非活性化 | アラートを非アクティブに変更 |
| | 未確認バッジ | ナビバーに未確認アラート件数をバッジ表示 |
| | リアルタイム通知 | WebSocketで新着アラートをリアルタイムに通知 |
| **アラートルール管理** | ルール作成 | マッチ条件、テンプレート、重要度を指定して作成（管理者操作） |
| | ルール更新 | 各フィールドの部分更新、有効/無効の切り替え（管理者操作） |
| | ルール削除 | 管理者のみ削除可能 |
| | 自動評価 | ログ作成時に有効ルールを評価し、条件一致でアラートを自動生成 |
| **タスク分類管理** | 分類一覧 | タスク分類マスタの一覧取得 |
| | 分類CUD | 管理者のみ作成・更新・削除可能 |
| **ユーザー管理** | ユーザー一覧 | 登録ユーザーの一覧取得 |
| | ユーザー作成 | メールアドレス、表示名、パスワードを指定して作成（管理者操作） |
| | ユーザー取得 | IDを指定してユーザー情報を取得 |
| | ユーザー編集 | 管理者は全フィールド編集可、一般ユーザーは自分の表示名のみ編集可 |
| | ユーザー削除 | 管理者のみ（自分自身の削除は不可） |
| | パスワード変更 | 自分のパスワードを現パスワード検証の上で変更 |
| | パスワードリセット | 管理者が他ユーザーのパスワードを強制リセット |
| **カレンダー管理** | イベント作成 | タイトル、日時、場所、会議室、繰り返しルールを指定して作成 |
| | イベント更新 | 各フィールドの部分更新、繰り返しイベントのスコープ指定（all/this/series） |
| | イベント削除 | 単一/シリーズ/以降を指定して削除 |
| | 参加者管理 | イベントへの参加者追加・削除・応答（accept/decline/tentative） |
| | リマインダー | イベント開始前の通知設定 |
| | 会議室管理 | 会議室の作成・更新・削除・空き状況確認（管理者操作） |
| | カレンダー設定 | デフォルト表示・色・リマインダー・表示項目のユーザー個別設定 |
| | グループフィルタ | サイドバーのグループドロップダウンでグループ単位のイベント表示切替 |
| **グループ管理** | グループ一覧 | グループの一覧取得 |
| | グループCUD | 管理者のみ作成・更新・削除可能 |
| | ユーザー所属 | ユーザーをグループに割り当て（ユーザー編集経由） |
| **Dashboard** | サマリー表示 | 各機能の概要をカード形式で一覧表示 |
| | 直近ログ | 直近5件のログを表示 |
| | 未完了Todo | 未完了Todoの上位5件を表示 |
| **国際化 (i18n)** | ロケール切替 | ユーザー設定の `preferred_locale`（ja/en）によるUI言語切替 |
| | テンプレート翻訳 | Jinja2 `_()` 関数によるサーバーサイド翻訳（gettext + Babel） |
| | JS翻訳 | `i18n.t()` 関数によるクライアントサイド翻訳（`static/locale/{locale}.json`） |
| | エラーメッセージ翻訳 | サービス層は英語キー、`app_error_handler` で境界層翻訳 |

## 2. 画面仕様

### 2.1 共通レイアウト

- ナビゲーションバー（ダークテーマ）に以下のリンクを配置:
  - Dashboard (`/`)
  - Todo (`/todos`) ※ナビバーから非表示（URLアクセスは可能）
  - Public Todos (`/todos/public`) ※ナビバーから非表示（URLアクセスは可能）
  - Task List (`/task-list`)
  - Attendance (`/attendance`)
  - Presence (`/presence`)
  - Tasks (`/tasks`)
  - Reports (`/reports`)
  - Summary (`/summary`)
  - Calendar (`/calendar`)
  - Logs (`/logs`)
  - Alerts (`/alerts`) ※未確認アラート件数バッジ付き
  - Users (`/users`)
- ナビゲーションバー右側にユーザーメニュー（ドロップダウン）:
  - ログインユーザーの表示名
  - Change Password（パスワード変更モーダル）
  - Logoutボタン
- 現在のパスに対応するナビゲーションリンクに `active` クラスが付与される。
- アラートバッジ: WebSocket (`/ws/alerts`) を監視し、新着時にバッジを自動更新。
- レスポンシブデザイン（Bootstrap 5のグリッドシステムを使用）。
- テンプレートファイル: `templates/base.html`
