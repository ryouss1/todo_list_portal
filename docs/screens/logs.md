# Logs画面 (`/logs`)

> 本ドキュメントは [spec_function.md](../spec_function.md) から分割された画面仕様です。

## ソースダッシュボード

- ログソースをテーブル形式で表示（状態、名前、グループ、方式、パス数、モード、ファイル数、最終スキャン、操作）
- テーブル行に以下を表示:
  - ステータスドット（ファイル変更検出: 赤、エラー: 赤、有効: 緑、無効: グレー）
  - ソース名 + ソース種別バッジ（WEB/HT/BATCH/OTHER）
  - グループ名
  - 接続方式バッジ（FTP/SMB）
  - 監視パス数
  - 収集モード（metadata_only/full_import）
  - ファイル数（全体 + new/updatedバッジ）
  - 最終チェック日時
  - 操作ボタン（View Files、Scan、Test Connection、Edit、Delete）
- **ファイル変更検出時の表示**（`has_alert=true` の場合）:
  - ソース行の下に赤背景の変更通知行を表示
  - 変更があったパスごとに**フォルダパス**を表示:
    - SMB: `\\{host}\{base_path}` 形式
    - FTP: `ftp://{host}:{port}{base_path}` 形式
  - 変更ファイル名の一覧を表示（新規/更新のバッジ付き）
  - フォルダパスをクリック → クリップボードにコピー（トースト通知でフィードバック）。コピーしたパスをExplorerのアドレスバーに貼り付けてフォルダに遷移可能
- 連続エラー時は次行にエラー詳細を赤背景で表示
- ソース登録モーダル（管理者のみ、タブ構成）:
  - 接続タブ: ソース名、グループ、接続方式（FTP/SMB）、ホスト、ポート、ユーザー名、パスワード、ドメイン（SMB時）
  - パスタブ: 監視パスの追加・削除（各行: ベースパス + ファイルパターン + 有効/無効 + 削除ボタン）
  - 設定タブ: エンコーディング、ソース種別、ポーリング間隔、収集モード、**ファイル変更通知チェックボックス**（`alert_on_change`）
  - パーサータブ: 正規表現パターン、severityフィールド、デフォルトseverity
- スキャン: テーブル行からスキャンを実行し、当日のファイルのタイムスタンプ変更を検出。結果モーダルに変更ファイル一覧とフォルダリンクを表示
- 接続テスト: テーブル行から接続テストを実行し、パスごとの結果（成功/失敗、検出ファイル数）を表示
- ファイル一覧: テーブル行から紐づくファイルの一覧を表示

## リアルタイムログストリーム（折り畳み可能）

- ログをテーブル形式で表示（受信日時、重要度、システム名、ログ種別、メッセージ）
- 重要度に応じた行の背景色:
  - ERROR / CRITICAL: 赤系背景
  - WARNING: 黄系背景
- 重要度バッジの色分け:
  - ERROR / CRITICAL: 赤
  - WARNING: 黄
  - DEBUG: グレー
  - INFO: 水色
- WebSocket接続状態バッジ（Connected: 緑、Disconnected: 赤）
- フィルターボタン: All / Important（WARNING以上のみ）
- WebSocket経由で新しいログを自動受信しリストの先頭に追加
- クライアント側のログ保持件数は最大200件
- WebSocket切断時は3秒後に自動再接続
