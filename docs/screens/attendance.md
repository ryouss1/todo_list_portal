# Attendance画面 (`/attendance`)

> 本ドキュメントは [spec_function.md](../spec_function.md) から分割された画面仕様です。

## 表示機能

- 現在の出勤状態をカード形式で表示
- 出勤中の場合、経過時間をリアルタイムでカウントアップ表示（HH:MM:SS形式）
- 出勤履歴をテーブル形式で表示（日付、出勤時刻、退勤時刻、勤務時間、メモ）
- 月別フィルタで表示期間を絞り込み
- 退勤未済の場合は「Active」バッジを表示

## 操作機能

- **Clock In**: 出勤ボタン（出勤中は無効化）
- **Clock Out**: 退勤ボタン（未出勤時は無効化）
- **Default Set**: プリセットから当日の勤怠を一括設定
- **手動登録（作成モーダル）**: モーダルダイアログで勤怠記録を手動作成
  - プリセットボタン: プリセット一覧から選択すると出退勤時刻・休憩時刻を自動入力
  - 休憩行の追加（最大3件）: 手動入力 or プリセットから自動入力
- **編集モーダル**: 既存の勤怠記録を修正
  - プリセットボタン: 「休憩をプリセットから入力」ボタンをクリックすると、ユーザーのデフォルトプリセットの休憩時刻を休憩行に自動入力（既存の休憩行を置換）
  - 休憩行の追加・削除（最大3件）
  - 出退勤時刻・メモの編集
- **Excelエクスポート**: 月別の勤怠データをダウンロード
- **月フィルタ**: `<input type="month">` の値変更と同時に `loadHistory()` を自動実行（ボタン押下不要）
- **今月に戻す**: 📅 ボタンをクリックすると月フィルタを当月にリセットし、履歴を再読み込み
- **前日未退勤ダイアログ**: `GET /api/attendances/status` が前日以前の未退勤レコードを検出した場合（`is_clocked_in=True` かつ `current_attendance.date < today`）、編集モーダルを自動表示して退勤時刻の入力を促す。退勤を記録するまで当日の出勤ボタンは無効
- メモの入力欄（任意）
