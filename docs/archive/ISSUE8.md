## 1. 背景

勤怠記録の**手動登録モーダル**にはプリセットボタン（出退勤時刻・休憩時刻の自動入力）が実装されているが、**編集モーダル**にはプリセットからの休憩時刻入力機能が存在しない。

編集時に休憩を追加する際、毎回手動で時刻を入力する必要があり、操作が煩雑になっている。

## 2. 現状

### 手動登録モーダル（createModal）
- プリセットボタン一覧が表示される
- ボタンクリックでプリセットの `clock_in`、`clock_out`、`break_start`、`break_end` を自動入力
- JS: `fillFromPreset()` → `addCreateBreakRow(p.break_start, p.break_end)`

### 編集モーダル（editModal）
- 出退勤時刻・メモの編集のみ
- 休憩行の追加・削除は可能（`addBreakRow()`）
- **プリセットからの休憩時刻入力機能なし** ← 今回の改善対象

## 3. 要件

### 3.1 機能概要

編集モーダルに「プリセットから休憩入力」ボタンを追加し、ユーザーのデフォルトプリセットの休憩時刻を自動入力できるようにする。

### 3.2 UI仕様

```
┌── 編集モーダル ──────────────────────────────┐
│ Clock In:  [09:00]   Clock Out: [18:00]      │
│ Note:      [___________]                     │
│                                              │
│ Breaks:                                      │
│  [12:00] - [13:00]  [×]                     │
│  + Add Break   [⚡ プリセット休憩]            │
│                                              │
│               [Cancel] [Save]                │
└──────────────────────────────────────────────┘
```

- 「+ Add Break」ボタンの横に「プリセット休憩」ボタンを配置
- ボタンクリックで以下の動作:
  1. 既存の休憩行をすべて削除（置換）
  2. ユーザーのデフォルトプリセットの `break_start`/`break_end` を新しい休憩行として追加
  3. プリセットに休憩が未設定の場合は「プリセットに休憩が設定されていません」トースト表示

### 3.3 動作仕様

| 条件 | 動作 |
|------|------|
| プリセットに休憩設定あり | 既存休憩行を置換し、プリセットの休憩時刻を入力 |
| プリセットに休憩設定なし | トースト警告を表示（「プリセットに休憩が設定されていません」） |
| ユーザーにデフォルトプリセット未設定 | プリセット一覧の先頭を使用（既存の `default_set` と同じロジック） |
| 休憩行が既に存在する | 確認なしで置換（プリセットからのデフォルト入力は明示的操作のため） |

### 3.4 既存の手動登録モーダルのプリセット機能との差異

| 項目 | 手動登録モーダル | 編集モーダル |
|------|----------------|-------------|
| プリセット選択 | 全プリセットをボタン一覧表示 | デフォルトプリセットのみ使用 |
| 入力対象 | 出退勤時刻 + 休憩時刻 | **休憩時刻のみ** |
| ボタン位置 | フォーム上部（独立セクション） | 休憩行の追加ボタン横 |

**理由**: 編集モーダルでは出退勤時刻は既に設定済みのため、休憩時刻のみの入力で十分。全プリセット選択UIは過剰。

## 4. 実装方針

### 4.1 フロントエンド変更

**ファイル: `templates/attendance.html`**
- 編集モーダルの「+ Add Break」ボタンの横に「プリセット休憩」ボタンを追加

**ファイル: `static/js/attendance.js`**
- `fillEditBreakFromPreset()` 関数を追加
  - `presets` 配列と `userPresetId` は既にグローバル変数として読み込み済み
  - 既存の休憩行を `#edit-breaks` からクリア
  - プリセットの `break_start`/`break_end` で `addBreakRow()` を呼び出し

### 4.2 バックエンド変更

**変更なし** — 既存の `PUT /api/attendances/{id}` API で休憩の更新に対応済み。

### 4.3 テスト

**変更なし** — フロントエンドUI変更のみのため、既存のAPIテストでカバー済み。
