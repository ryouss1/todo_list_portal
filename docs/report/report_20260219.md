## 1. ISSUE-075: summary_service.py 単一パスリファクタリング

### 概要

`app/services/summary_service.py` の `reports` リストに対して行っていた 8 回の独立したイテレーションを、1 回のループに統合した。

### 変更内容

**Before**: 8 つの別々の `for r in reports` / Counter 内包表記
- `user_report_counts`: 全件スキャン × 1
- `user_has_today`: 全件スキャン × 1
- `user_cat_*` (2変数): 全件スキャン × 1
- `date_counts`: 全件スキャン × 1
- `date_cat_*` (2変数): 全件スキャン × 1
- `cat_stats`, `issues`, `recent_reports_raw`: 全件スキャン × 1

**After**: 1 つのループで全変数を同時に集計
```python
for r in reports:
    user_report_counts[r.user_id] += 1
    if r.report_date == today:
        user_has_today.add(r.user_id)
    user_cat_counts[r.user_id][r.category_id] += 1
    ...
```

### 成果

- 計算量: O(8n) → O(n)（日報件数 n に対して8パスが1パスに）
- ISSUE5.md 更新: Medium 5→4、総件数 11→10、解消済 18→19

## 2. Task List 担当者設計（ISSUE: 強制自己アサイン バグ）

### 概要

タスクリストで新規アイテム作成時に担当者が常に自分になり、かつ解除できないバグを発見・設計。

### バグ原因

`static/js/task_list.js` にて:
```javascript
// 新規作成時に強制的に自分にアサイン → バグ
data.assignee_id = currentUserId;
await api.post('/api/task-list/', data);
```

加えて `TaskListItemUpdate` に `assignee_id` フィールドがなく、編集モーダルから担当者を変更できなかった。

### 設計書作成

`docs/api/task-list/DESIGN_task-list_assignee.md` を作成し、3 案を比較：

| 案 | 内容 | 採用 |
|----|------|------|
| A | デフォルト未割当（セレクタなし） | - |
| B | デフォルト自分（セレクタなし） | - |
| **C** | **モーダルに担当者セレクタ追加、デフォルト未割当** | **✅ 採用** |

**採用理由**: チーム共有バックログの pull-based ワークフロー（未割当プール → 担当決定 → 着手）に合致。
